Monopoly game project

I am building a Monopoly-style project because it forces a lot of useful engineering problems into one place: turns, rules, money, ownership, player state, and edge cases that have to stay consistent.

Focus
Game state, rule modeling, UI flow
What I learned
The biggest lesson has been separating the rules from the screen. When the game logic is its own thing, it becomes much easier to test, debug, and trust the UI that sits on top of it.
